# Code of Conduct / 行動規範

## English

### Our commitment

We want Batonel to be a respectful, welcoming, and constructive project for contributors, users, and maintainers.

Participation in this community should be safe and professional regardless of experience level, age, body size, disability, ethnicity, gender identity or expression, level of experience, nationality, personal appearance, race, religion, or sexual identity and orientation.

### Our standards

Examples of behavior that contributes to a positive environment include:

- being respectful and patient
- giving constructive feedback
- assuming good intent while staying specific
- focusing discussion on ideas, trade-offs, and implementation details
- helping others learn and participate

Examples of unacceptable behavior include:

- harassment, intimidation, or discriminatory language
- personal attacks, insults, or trolling
- repeated bad-faith arguments or deliberate disruption
- publishing someone else’s private information without permission
- sexualized language or unwelcome sexual attention
- dismissive or hostile behavior that makes collaboration unsafe

### Scope

This Code of Conduct applies within all Batonel project spaces, including issues, pull requests, discussions, chat spaces, and other project-managed communication channels.

### Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct.
They may remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct.

### Reporting

If you experience or witness unacceptable behavior, please report it to the maintainers.

Replace this section with your actual contact information before publishing, for example:

- project email: `maintainers@example.com`
- GitHub security or contact address
- organization contact form

All reports should be handled with care and respect for the reporter’s privacy.

### Consequences

Maintainers may take any action they deem appropriate, including warning contributors, removing content, temporarily restricting participation, or banning repeat offenders from the project spaces.

### Attribution

## 日本語

### 私たちの姿勢

Batonel は、コントリビューター、利用者、メンテナーのすべてにとって、敬意があり、参加しやすく、建設的な OSS であることを目指します。

経験年数、年齢、体格、障害、民族性、性自認や性表現、国籍、外見、人種、宗教、性的指向などにかかわらず、このコミュニティへの参加が安全でプロフェッショナルなものであることを大切にします。

### 望ましい行動

次のような行動を歓迎します。

- 敬意と落ち着きを持ってやり取りする
- 建設的で具体的なフィードバックを行う
- 相手の善意を前提にしつつ、論点は明確にする
- 人ではなく、アイデア・設計・実装上の論点に集中する
- 他の参加者が学び、参加しやすい環境をつくる

### 受け入れられない行動

次のような行動は受け入れられません。

- ハラスメント、威圧、差別的表現
- 個人攻撃、侮辱、煽り行為
- 悪意ある反復的な議論妨害
- 本人の許可なく個人情報を公開すること
- 性的な表現や望まれない性的関心の表明
- 協働を困難にする敵対的または見下すような態度

### 適用範囲

この行動規範は、Batonel プロジェクトに関係する Issue、Pull Request、Discussion、チャット、その他プロジェクト管理下のコミュニケーション空間に適用されます。

### 運用と対応

メンテナーは、この行動規範を説明し、必要に応じて適用する責任を持ちます。
違反があった場合、コメント、コミット、コード、Issue、その他の投稿を編集・削除・拒否することがあります。

### 報告方法

不適切な行動を受けた場合、または目撃した場合は、メンテナーへ報告してください。

公開前に、以下のような実際の連絡先へ置き換えてください。

- project email: `maintainers@example.com`
- GitHub 上の連絡先
- 組織の問い合わせフォーム

報告内容は、報告者のプライバシーに配慮して慎重に扱います。

### 想定される対応

メンテナーは、必要に応じて、注意、投稿削除、一時的な参加制限、継続的な違反者の排除などの対応を行うことがあります。

### 参考

この文書は Contributor Covenant v2.1 の考え方を参考にしつつ、Batonel 向けに簡潔化したものです。
